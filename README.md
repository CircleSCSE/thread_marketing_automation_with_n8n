# [기록] 반응하는 인간 군상: AI 자동화로 돈이 벌리는가?

이 프로젝트는 n8n과 AI(Gemini)를 활용해 콘텐츠를 자동으로 올리고, 실제로 사람들이 지갑을 여는지 17일간 몸으로 부딪히며 확인한 과정입니다. 심리학 책에서 본 기법들 ( 닻 내림 효과, 단순 노출 효과 등 )을 실제로 적용해보고자 하는 의도도 있었습니다.

무엇보다, 장기간 하나의 SNS 계정을 운영하면 그냥 오래 꾸준히 한 것 자체로 효과가 있을까? 싶은 생각도 들었기에 시작했습니다. 제가 직접 했을 때는 꾸준하게 매일 하는 것은 2주일도 못 갔기 때문에, 그냥 변주를 주면서 글을 올리는 자동화를 만들어두고, 한 6개월 까먹고 있다가 나중에 결과를 보고 싶다는 욕구가 있었습니다.

제가 구축한 가상의 철학 이론을 바탕으로 글을 자동으로 생성하는 프롬프트를 구축하였고, 다분화하였습니다.

## 🛠 고친 것들과 시도한 것들

### 1. "노트북 끄고 싶다" (서버 구축)

처음엔 노트북 켜놓을 때만 돌아갔는데, 오라클 무료 서버에 n8n을 올려서 **24시간 자동화**로 바꿨습니다. 오라클 자체 방화벽 해제 설정 같은 짜증 나는 문제들이 많았지만 4시간 동안 붙잡고 해결했습니다.

### 2. "AI 티 안 나게 만들기"

AI가 너무 로봇처럼 말하면 사람들이 바로 눈치챕니다. 그래서 다음 세 가지를 적용했습니다.

- **말투 다변화**: 에이전트 노드를 3개 쓰고 '답글 패턴'을 나눠서 인간적인 느낌을 섞었습니다.
- **유연함 조절**: 창의성 수치(Temperature)를 **0.8** 정도로 높여서 AI가 좀 더 유도리 있게 글을 쓰게 만들었습니다. ( 이는 해커톤을 통해 얻은 Upstage API에서 했습니다. gemini는 좀 더 성능이 좋아서 할 필요가 없었습니다. )
- **썸네일 ‘nano banana’ 이용**: 글만 있으면 안 읽으니까, postype에 AI 프롬프트로 생성한 이미지를 넣어서 시각적인 요소를 채워넣었습니다.

### 3. "규제와 밴(Ban) 대응"

글이 너무 거칠거나 특정 단어(예: 욕설, 모욕, 팔로우 유도 등)가 반복되면 플랫폼에서 노출을 막아버립니다.

- 너무 적나라한 표현은 순화하도록 프롬프트를 고쳤고, 절대로 이상한 출력은 못하도록 규제 프롬프트를 넣었습니다.
