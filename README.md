# [기록] 반응하는 인간 군상: AI 자동화로 돈이 벌리는가?

이 프로젝트는 n8n과 AI(Gemini)를 활용해 콘텐츠를 자동으로 올리고, 실제로 사람들이 지갑을 여는지 17일간 몸으로 부딪히며 확인한 과정입니다. 심리학 책에서 본 기법들(닻 내림 효과, 단순 노출 효과 등)을 실제로 적용해보고자 하는 의도도 있었습니다.

무엇보다, "장기간 하나의 SNS 계정을 꾸준히 운영하면 효과가 있을까?"라는 의문에 대한 답을 찾고 싶었습니다. 제 성향상 수동으로 꾸준히 하는 것은 불가능했기에, 변주를 주며 자동으로 글이 올라가는 시스템을 구축하고 잊고 지내다 나중에 결과를 확인하고 싶었습니다.

제가 구축한 가상의 철학 이론을 바탕으로 글을 자동으로 생성하는 프롬프트를 구축하였고, 이를 다분화하여 적용했습니다.

## 🛠 고친 것들과 시도한 것들

### 1. "노트북 끄고 싶다" (서버 구축)

처음엔 노트북 켜놓을 때만 돌아갔는데, 오라클 무료 서버에 n8n을 올려서 **24시간 자동화**로 바꿨습니다. 오라클 자체 방화벽 해제 설정 등 성가신 문제들이 많았지만 4시간 만에 해결했습니다.

### 2. "AI 티 안 나게 만들기"

AI가 로봇처럼 말하면 사람들이 바로 눈치챕니다. 다음 세 가지를 적용해 인간미를 더했습니다.

- **말투 다변화**: 에이전트 노드를 나누고 '답글 패턴'을 분산시켜 기계적인 느낌을 줄였습니다.
- **유연함 조절**: 창의성 수치(Temperature)를 **0.8**로 상향 조정하여 글의 유연성을 확보했습니다. (Upstage API 활용 경험 기반, Gemini는 기본 성능이 우수하여 별도 조정 불필요)
- **시각 요소 추가**: 텍스트만 있는 지루함을 탈피하기 위해, 프롬프트로 생성한 이미지(nano banana 스타일)를 썸네일로 자동 삽입했습니다.

### 3. "규제와 밴(Ban) 대응"

욕설, 모욕, 노골적인 팔로우 유도 등은 플랫폼 노출 정지의 원인이 됩니다.

- 표현 수위를 순화하도록 기본 프롬프트를 수정하고, 시스템 레벨에서 유해한 출력을 차단하는 **규제 프롬프트 레이어**를 추가했습니다.
